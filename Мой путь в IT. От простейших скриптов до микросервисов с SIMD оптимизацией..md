Перейдём к темам:
1. [[Изучение языка С. От простых концепций до asm, simd и встраивания в другие языки.]] Советую начать именно с Си, если вы не опытен в разработке - низкоуровневое понимание кода и его работы даёт огромный простор для действий и оптимизаций. Прекрасный пример - моё общение с TechLead компании X5Tech. Во время диалога чётко уяснил для себя, что просто писать хороший бэк\фронт\скрипты - мало. Нужно делать так, чтобы они работали на 1000+ RPS, а обычными инструментами Python - этого не достичь. Вот они и стали использовать PyPy (JIT-компиляция, почитайте для понимания). Без Си, особенно на языке где компилятор и большая часть расширений написаны на нём, можно будет максимум написать демо-версию EDA-системы с брокером и БД. И то не факт, что вы поймёте всё что написали (Особенно если с ИИ). Как итог - Си это основа работы ИТ в нашем мире. Писать на нём - избыточно почти во всех случаях, но понимать что он из себя представляет - очень желательно.
2. [[Ядра операционных систем. Искусство низкоуровневой оптимизации в купе с кросс-платформенностью.]] Для тех, кто пишет платформозависимый код. Разберём ядра ОС, посмотрим как кто работает, и что делать чтобы код работал быстрее.
3. [[C-threading. Полное руководство от начала до продвинутых концепций.]] Этот раздел уже для тех, кто проникся первым пунктом и готов уделить время нашему старичку. По-сути - избыточно для современного программиста, но почему бы и не изучить, как работает многопоток по-настоящему?
4. [[Основы python для продвинутых пользователей]]. Да-да, именно для продвинутых. Чтобы читать - нужно иметь хоть какой-то опыт работы с прикладным инструментом, иначе - не лезь убьёт! А для чего нужен сам python? Удобный и функциональный клей, который совместим с Си и стабильно справляется с web-задачами и скриптами. Самое то для автоматизации, быстрых малых API и интерфейсов взаимодействия с пользователем. Если совсем упороться - можно и ML на нём попробовать, но это уже на любителя.
5. [[Python-NOGIL. Установка-внедрение-использование.]] Продвинутый пункт для тех, кто хочет реальный threading на python без костылей.
6. [[Cython. Клей между пайтоном и C для максимальной мощи в обработке огромных данных.]] Вот это уже крайне неоднозначный пункт. Подойдёт тем, кто не хочет учить Rust, знает Си и Python, и кому нужна минимальная задержка между вызовами функций расширений (Если что, PyO3 на текущий момент имеет чуть большие накладные расходы на вызов функции в Python - сам по себе Rust быстрее Cython, но для High RPS лучше использовать то что даёт меньше задержки). Так же будет малое дополнение про основы Rust и работе с PyO3. Будет крайне полезно для тех, кто хочет современный стек и быстрый Python.
