# auto

**Категория:** Класс хранения

## Назначение

Является классом хранения данных в области памяти программы (Стеке). Автоматическое выделение памяти для локальных переменных. 

Переменные со спецификатором `auto` создаются при входе в блок и уничтожаются при выходе. Нельзя использовать для параметров функции. Идеален для быстрого определения типа переменной компилятором. 

Все переменные auto хранятся непосредственно в стеке и не видимы вне своего { } блока. 

Является устаревшим и избыточным для современного C\C++ стандартом, так как все локальные переменные - автоматические. 

## Особенности

- Используется **по умолчанию** для всех локальных переменных
- Явное указание `auto` избыточно (редко используется)
- Переменные `auto` имеют **неопределённое значение** по умолчанию (не инициализируются)

## Пример

```c
void function() {
    auto int x = 10;  // явное указание (избыточно)
    int y = 20;       // неявно auto

    // x и y уничтожаются при выходе из функции
}
```

## Пример из практики

Сравнение `auto` и `static` в [[Auto vs Static]]:

```c
void count() {
    static int x = 0;  // сохраняется между вызовами
    auto int y = 0;    // сбрасывается при каждом вызове

    x++;  // 1, 2, 3...
    y++;  // всегда 1

    printf("%d, %d\n", x, y);  // 1,1  2,1  3,1
}
```

**Вывод:**
- `static` переменная `x` сохраняет значение между вызовами функции
- `auto` переменная `y` инициализируется заново при каждом вызове
