# Статические библиотеки

Статические библиотеки (.a) — архивы объектных файлов, включаемые в исполняемый файл.

## Создание статической библиотеки

### 1. Компиляция объектных файлов

```bash
gcc -c utils.c -o utils.o
gcc -c math_ops.c -o math_ops.o
gcc -c io.c -o io.o
```

### 2. Создание архива

```bash
ar rcs libmylib.a utils.o math_ops.o io.o
```

**Флаги ar:**
- `r` — вставить/заменить файлы
- `c` — создать архив
- `s` — создать индекс символов

### 3. Использование библиотеки

```bash
gcc main.c -L. -lmylib -o program
```

## Пример полного цикла

```c
// utils.h
#ifndef UTILS_H
#define UTILS_H
void print_msg(const char* msg);
int add(int a, int b);
#endif
```

```c
// utils.c
#include "utils.h"
#include <stdio.h>

void print_msg(const char* msg) {
    printf("%s\n", msg);
}

int add(int a, int b) {
    return a + b;
}
```

```c
// main.c
#include "utils.h"

int main() {
    print_msg("Hello");
    int result = add(2, 3);
    return 0;
}
```

```bash
# Создание библиотеки
gcc -c utils.c -o utils.o
ar rcs libutils.a utils.o

# Использование
gcc main.c -L. -lutils -o program
./program
```

## Просмотр библиотеки

```bash
# Содержание архива
ar -t libmylib.a

# Подробная информация
ar -tv libmylib.a

# Символы в библиотеке
nm libmylib.a

# Извлечение объектов
ar -x libmylib.a
```

## Модификация библиотеки

```bash
# Добавить объект
ar r libmylib.a newobj.o

# Заменить объект
ar r libmylib.a updated.o

# Удалить объект
ar d libmylib.a oldobj.o

# Пересоздать индекс
ar s libmylib.a
```

## Преимущества статических библиотек

| Плюс | Описание |
|------|----------|
| Переносимость | Все зависимости внутри |
| Скорость | Нет динамической загрузки |
| Версионирование | Конкретная версия в файле |

## Недостатки

| Минус | Описание |
|-------|----------|
| Размер | Код дублируется в каждом файле |
| Обновления | Нужно пересобирать для обновлений |
| Память | Каждая программа загружает свою копию |

## Статическая линковка системы

```bash
# Полностью статический исполняемый файл
gcc -static main.c -o program

# Проверка
ldd program  # "not a dynamic executable"
```

## Именование библиотек

```
lib<имя>.a
```

**Примеры:**
- `libutils.a` → `-lutils`
- `libmylib.a` → `-lmylib`
