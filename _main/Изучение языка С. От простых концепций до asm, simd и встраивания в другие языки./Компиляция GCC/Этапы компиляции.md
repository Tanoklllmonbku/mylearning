# Этапы компиляции

Исходный код C проходит 4 этапа перед получением исполняемого файла.

## 4 этапа компиляции

```
.c файл → Препроцессинг → .i файл → Компиляция → .s файл → 
Ассемблирование → .o файл → Линковка → исполняемый файл
```

## 1. Препроцессинг

**Что делает:**
- Обрабатывает директивы (`#include`, `#define`, `#if`)
- Удаляет комментарии
- Раскрывает макросы

**Команда:**
```bash
gcc -E program.c -o program.i
```

**Пример:**
```c
// program.c
#include <stdio.h>
#define MAX 100

int main() {
    printf("%d\n", MAX);
    return 0;
}
```

```bash
gcc -E program.c -o program.i
head program.i  # просмотр результата
```

## 2. Компиляция

**Что делает:**
- Преобразует препроцессированный код в ассемблерный
- Выполняет оптимизации
- Проверяет типы

**Команда:**
```bash
gcc -S program.i -o program.s
# или сразу из .c
gcc -S program.c -o program.s
```

**Пример:**
```bash
gcc -S program.c -o program.s
cat program.s  # ассемблерный код
```

## 3. Ассемблирование

**Что делает:**
- Преобразует ассемблерный код в машинный
- Создаёт объектный файл (.o)

**Команда:**
```bash
gcc -c program.s -o program.o
# или сразу из .c
gcc -c program.c -o program.o
```

**Результат:**
- `program.o` — объектный файл (машинный код + символы)

## 4. Линковка (компоновка)

**Что делает:**
- Объединяет объектные файлы
- Разрешает внешние символы
- Добавляет библиотеки
- Создаёт исполняемый файл

**Команда:**
```bash
gcc program.o -o program
```

## Просмотр всех этапов

```bash
# Один файл: все этапы сразу
gcc program.c -o program

# Сохранение промежуточных файлов
gcc -save-temps program.c -o program
# Создаст: program.i, program.s, program.o, program
```

## Многофайловая компиляция

```bash
# Раздельная компиляция
gcc -c main.c -o main.o
gcc -c utils.c -o utils.o
gcc main.o utils.o -o program

# Или одной командой
gcc main.c utils.c -o program
```
