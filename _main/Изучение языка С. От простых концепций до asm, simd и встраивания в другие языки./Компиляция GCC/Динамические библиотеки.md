# Динамические библиотеки

Динамические библиотеки (.so) — разделяемые библиотеки, загружаемые во время выполнения.

## Создание динамической библиотеки

### 1. Компиляция с -fPIC

```bash
gcc -fPIC -c utils.c -o utils.o
gcc -fPIC -c math_ops.c -o math_ops.o
```

**-fPIC** — позиционно-независимый код (Position Independent Code)

### 2. Создание .so

```bash
gcc -shared utils.o math_ops.o -o libmylib.so
```

### 3. Использование

```bash
gcc main.c -L. -lmylib -o program
```

## Пример полного цикла

```c
// utils.h
#ifndef UTILS_H
#define UTILS_H
void print_msg(const char* msg);
int add(int a, int b);
#endif
```

```c
// utils.c
#include "utils.h"
#include <stdio.h>

void print_msg(const char* msg) {
    printf("%s\n", msg);
}

int add(int a, int b) {
    return a + b;
}
```

```c
// main.c
#include "utils.h"

int main() {
    print_msg("Hello");
    int result = add(2, 3);
    return 0;
}
```

```bash
# Создание библиотеки
gcc -fPIC -c utils.c -o utils.o
gcc -shared utils.o -o libutils.so

# Компиляция программы
gcc main.c -L. -lutils -o program

# Запуск (нужно указать путь к .so)
LD_LIBRARY_PATH=. ./program
```

## Установка библиотеки в систему

```bash
# Копирование в системную директорию
sudo cp libutils.so /usr/lib/
sudo ldconfig  # обновление кэша

# Или в /usr/local/lib
sudo cp libutils.so /usr/local/lib/
sudo ldconfig
```

## Проверка зависимостей

```bash
# Показать используемые .so
ldd program

# Вывод:
# libutils.so => ./libutils.so
# libc.so.6 => /lib/x86_64-linux-gnu/libc.so.6
```

## Пути поиска библиотек

1. Переменная `LD_LIBRARY_PATH`
2. Кэш `/etc/ld.so.cache`
3. Стандартные пути `/lib`, `/usr/lib`

```bash
# Временный путь
export LD_LIBRARY_PATH=/path/to/libs:$LD_LIBRARY_PATH

# Постоянный путь
echo "/path/to/libs" | sudo tee /etc/ld.so.conf.d/mylib.conf
sudo ldconfig
```

## Версионирование библиотек

```bash
# Создание с версией
gcc -shared -Wl,-soname,libutils.so.1 -o libutils.so.1.0 utils.o

# Символические ссылки
ln -sf libutils.so.1.0 libutils.so.1
ln -sf libutils.so.1 libutils.so
```

## Преимущества динамических библиотек

| Плюс | Описание |
|------|----------|
| Размер | Код разделяется между программами |
| Обновления | Можно обновлять без пересборки |
| Память | Одна копия в памяти |

## Недостатки

| Минус | Описание |
|-------|----------|
| Зависимости | Нужны внешние .so файлы |
| Совместимость | Проблемы версий (DLL hell) |
| Загрузка | Медленнее запуск |

## Сравнение со статическими

| Характеристика | Статическая (.a) | Динамическая (.so) |
|----------------|------------------|-------------------|
| Размер файла | Больше | Меньше |
| Зависимости | Нет | Нужны .so |
| Обновления | Пересборка | Замена .so |
| Память | Копия на программу | Общая |
