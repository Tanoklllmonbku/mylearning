# Флаги линковки

Флаги GCC для компоновки (линковки) программ.

## Базовая линковка

```bash
gcc main.o utils.o -o program
```

## -l: Подключение библиотек

```bash
# Математическая библиотека
gcc main.c -lm -o program

# Потоки (pthread)
gcc main.c -lpthread -o program

# Несколько библиотек
gcc main.c -lm -lpthread -o program
```

### Распространённые библиотеки

| Флаг | Библиотека |
|------|------------|
| `-lm` | math (математика) |
| `-lpthread` | pthread (потоки) |
| `-ldl` | dl (динамическая загрузка) |
| `-lrt` | rt (real-time) |

## -L: Пути к библиотекам

```bash
# Поиск в директории
gcc -L./lib main.c -lmylib -o program

# Несколько путей
gcc -L./lib -L/usr/local/lib main.c -lmylib -o program
```

## -I: Пути к заголовкам

```bash
gcc -I./include main.c -o program
```

## -shared: Динамическая библиотека

```bash
# Создание .so
gcc -shared -fPIC utils.c -o libutils.so

# Использование
gcc main.c -L. -lutils -o program
```

## -static: Статическая линковка

```bash
gcc -static main.c -o program
```

Все библиотеки включаются в исполняемый файл.

**Преимущества:**
- Переносимый файл
- Не нужны внешние .so

**Недостатки:**
- Большой размер
- Нет обновлений библиотек

## -fPIC: Позиционно-независимый код

```bash
# Для динамических библиотек
gcc -fPIC -c utils.c -o utils.o
gcc -shared utils.o -o libutils.so
```

## Порядок флагов важен!

```bash
# ПРАВИЛЬНО: библиотеки после исходников
gcc main.c -lm -o program

# НЕПРАВИЛЬНО: библиотека до файла
gcc -lm main.c -o program  # может не найти символы
```

## -Wl: Передача флагов линкеру

```bash
# Указать путь поиска во время выполнения
gcc -Wl,-rpath,/usr/local/lib main.c -o program

# Несколько путей
gcc -Wl,-rpath,/usr/local/lib,-rpath,/opt/lib main.c -o program
```

## -o: Имя выходного файла

```bash
gcc main.c -o myapp
```

## Пример: полный проект

```bash
# Структура
# project/
#   src/main.c
#   src/utils.c
#   include/utils.h
#   lib/libcustom.a

# Компиляция
gcc -I./include -c src/main.c -o main.o
gcc -I./include -c src/utils.c -o utils.o

# Линковка
gcc main.o utils.o -L./lib -lcustom -lm -o program
```

## Проверка зависимостей

```bash
# Показать используемые библиотеки
ldd program

# Показать символы
nm program
objdump -T program
```
