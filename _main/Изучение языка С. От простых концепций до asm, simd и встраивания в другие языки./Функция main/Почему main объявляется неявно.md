# Почему `main` объявляется неявно

## Неявное объявление функции `main`

Функция `main` **не имеет явного объявления** в стандартной библиотеке C. Программа не должна и не может предоставлять прототип для `main`:

```c
// ❌ НЕЛЬЗЯ: программа не должна объявлять прототип main
int main(int argc, char *argv[]);  // Бессмысленно

// ✅ МОЖНО: только определение
int main(int argc, char *argv[]) {
    return 0;
}
```

## Причины неявного объявления

### 1. Специальная роль в языке

`main` — это не обычная функция, а **точка входа**, определяемая стандартом C:

> «The function called at program startup is named `main`. The implementation declares no prototype for this function.»

Стандарт явно указывает:
- Реализация **не объявляет прототип** для `main`
- Программа должна **предоставить определение**
- Сигнатура определяется стандартом, а не заголовочным файлом

### 2. Среда выполнения (Runtime)

Среда выполнения C (CRT) знает ожидаемую сигнатуру `main` на этапе компиляции:

```
┌─────────────────────────────────────┐
│   Среда выполнения (CRT)            │
│                                     │
│   Ожидает: main(int, char**)        │
│   или: main(void)                   │
│                                     │
│   Вызов: call _main                 │
└─────────────────────────────────────┘
```

Компилятор и линковщик:
- **Знают** ожидаемую сигнатуру `main`
- **Проверяют** соответствие определения стандарту
- **Не требуют** явного прототипа

### 3. Отсутствие в заголовочных файлах

В отличие от других функций, `main` не объявляется в заголовочных файлах:

| Функция | Заголовок | Объявление |
|---------|-----------|------------|
| `printf` | `<stdio.h>` | `int printf(const char*, ...);` |
| `malloc` | `<stdlib.h>` | `void *malloc(size_t);` |
| `main` | **Нет** | Не объявляется |

```c
#include <stdio.h>    // Объявления для printf, scanf...
#include <stdlib.h>   // Объявления для malloc, exit...
// Но НЕ для main!
```

## Механизм работы

### Этапы компиляции

1. **Препроцессинг**: обработка директив `#include`, `#define`
2. **Компиляция**: проверка синтаксиса, семантики
3. **Сборка**: создание объектного файла
4. **Линковка**: связывание с CRT, поиск `main`

```
Исходный код → [Компилятор] → Объектный файл → [Линковщик] → Исполняемый файл
                     ↓                              ↓
              Проверка main                  Поиск символа _main
              (соответствие стандарту)       (точка входа)
```

### Проверка компилятором

Компилятор проверяет определение `main` на соответствие стандарту:

```c
// ✅ Допустимые формы
int main(void) { }
int main(int argc, char *argv[]) { }
int main(int argc, char *argv[], char *envp[]) { }  // Расширение

// ❌ Недопустимые формы
void main(void) { }           // Нестандартно (MSVC допускает)
int main(int argc) { }        // Неполная сигнатура
static int main(void) { }     // Internal linkage
inline int main(void) { }     // Inline запрещено
```

## Почему это важно

### 1. Предотвращение ошибок

Если бы `main` объявлялась явно, программисты могли бы:
- Объявить неправильную сигнатуру
- Использовать несовместимый тип возврата
- Создать конфликт с реальной точкой входа

```c
// ❌ Если бы было возможно:
int main(char *argv[], int argc);  // Неправильный порядок!
```

### 2. Унификация

Неявное объявление гарантирует:
- **Единую сигнатуру** для всех программ
- **Совместимость** со средой выполнения
- **Переносимость** между компиляторами

### 3. Специальный статус

`main` имеет уникальный статус в языке C:
- Не обычная функция, а **точка входа**
- Определяется стандартом, а не библиотекой
- Вызывается средой выполнения, а не кодом программы

## Сравнение с другими функциями

| Функция | Объявление | Кто вызывает |
|---------|------------|--------------|
| `printf` | `<stdio.h>` | Код программы |
| `exit` | `<stdlib.h>` | Код программы |
| `main` | **Не объявляется** | Среда выполнения |

## Стандарт C

Стандарт C явно определяет специальный статус `main`:

> «It shall be defined with a return type of `int` and with no parameters, or with two parameters... The implementation declares no prototype for this function.»

**Разделы стандарта:**
- **C23**: §5.1.2.2.1
- **C17**: §5.1.2.2.1
- **C11**: §5.1.2.2.1
- **C99**: §5.1.2.2.1

## Пример

```c
// ✅ Правильная программа
#include <stdio.h>

// Не нужно объявлять main!
int main(int argc, char *argv[]) {
    printf("Program started\n");
    return 0;
}

// ❌ Бессмысленное объявление (не требуется)
// int main(int argc, char *argv[]);
```

## Вывод

`main` объявляется неявно потому что:
1. Это **специальная функция** — точка входа, определяемая стандартом
2. Среда выполнения **знает сигнатуру** без явного объявления
3. Это предотвращает **ошибки сигнатуры** и обеспечивает унификацию
4. Стандарт C явно требует **отсутствия прототипа**
