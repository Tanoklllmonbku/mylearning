# Завершение программы и передача управления

## Способы завершения программы

Программа на C может завершиться несколькими способами:

| Способ | Механизм | Тип завершения |
|--------|----------|----------------|
| `return` из `main` | Возврат из функции | Нормальное |
| Конец функции `main` | Неявный возврат | Нормальное |
| `exit()` | Вызов функции | Нормальное |
| `_Exit()` | Немедленное завершение | Нормальное (без очистки) |
| `quick_exit()` | Быстрое завершение (C11) | Нормальное (мин. очистка) |
| `abort()` | Аварийное завершение | Ненормальное |

## Возврат из `main`

### Явный `return`

Возврат из `main` передаёт управление среде выполнения:

```c
int main(void) {
    // Код программы
    return 0;  // Передача управления в CRT
}
```

**Эквивалентность `exit`:**
Возврат из начального вызова `main` эквивалентен вызову:

```c
exit(возвращаемое_значение);
```

### Неявный возврат (C99 и выше)

Начиная с **C99**, если управление достигает конца `main`, неявно выполняется `return 0`:

```c
// C99+: эквивалентно return 0;
int main(void) {
    printf("Hello\n");
} // Неявный return 0;
```

**До C99:** статус возврата был неопределён при отсутствии `return`.

## Поток управления при завершении

### Диаграмма потока

```
┌─────────────────────────────────────────────────────────────┐
│                    Среда выполнения (CRT)                   │
│                                                             │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────────┐ │
│  │  _start     │───▶│  main()     │───▶│  exit(status)   │ │
│  │  (точка     │    │  (программа)│    │  (очистка)      │ │
│  │   входа)    │    │             │    │                 │ │
│  └─────────────┘    └──────┬──────┘    └────────┬────────┘ │
│                            │                     │         │
│                            │ return              │         │
│                            ▼                     ▼         │
│                     ┌─────────────────────────────────┐    │
│                     │  Завершение программы           │    │
│                     │  - atexit обработчики           │    │
│                     │  - Закрытие потоков             │    │
│                     │  - Сброс буферов                │    │
│                     │  - Возврат в ОС                 │    │
│                     └─────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

### Этапы завершения через `return`

1. **Возврат из `main`** — выполнение `return value;`
2. **Вызов `exit(value)`** — среда выполнения вызывает `exit`
3. **Обработчики `atexit`** — вызываются в обратном порядке регистрации
4. **Очистка потоков** — сброс буферов, закрытие файлов
5. **Удаление временных файлов** — созданных через `tmpfile()`
6. **Возврат в ОС** — передача кода завершения

## Передача управления в `main`

### От среды выполнения к программе

Среда выполнения (CRT) передаёт управление в `main` следующим образом:

```c
// Псевдокод CRT
void _start(void) {
    // 1. Инициализация
    init_static_objects();      // Статические объекты
    init_runtime();             // Среда выполнения
    
    // 2. Подготовка аргументов
    int argc = get_argc();
    char **argv = get_argv();
    char **envp = get_envp();
    
    // 3. Вызов main
    int result = main(argc, argv, envp);
    
    // 4. Завершение
    exit(result);
}
```

### Инициализация перед `main`

Перед вызовом `main` выполняются:

1. **Инициализация статических объектов**
   ```c
   static int x = 42;  // Инициализируется до main
   ```

2. **Настройка потоков stdio**
   ```c
   // stdin, stdout, stderr становятся доступными
   ```

3. **Установка локали**
   ```c
   // Базовая локаль "C"
   ```

## Рекурсивный вызов `main`

### Запрет рекурсии

Стандарт C **запрещает** рекурсивный вызов `main`:

```c
// ❌ ЗАПРЕЩЕНО: рекурсивный вызов main
int main(int argc, char *argv[]) {
    if (argc > 1) {
        main(argc - 1, argv);  // Ошибка: нельзя вызывать main
    }
    return 0;
}
```

**Причины:**
- `main` — точка входа, а не обычная функция
- Возврат из рекурсивного вызова не определён
- Среда выполнения ожидает единственный вызов

### Получение адреса `main`

Запрещено получение адреса функции `main`:

```c
// ❌ ЗАПРЕЩЕНО
int (*fp)(int, char**) = &main;  // Ошибка компиляции
fp(argc, argv);                   // Нельзя вызвать main
```

## Коды завершения

### Стандартные коды

| Код | Макрос | Значение |
|-----|--------|----------|
| `0` | `EXIT_SUCCESS` | Успешное завершение |
| `1` | `EXIT_FAILURE` | Неуспешное завершение |
| `2-255` | — | Реализуемо-зависимые коды |

### Пример использования

```c
#include <stdio.h>
#include <stdlib.h>

int main(int argc, char *argv[]) {
    if (argc < 2) {
        fprintf(stderr, "Ошибка: требуется аргумент\n");
        return EXIT_FAILURE;  // Возврат 1
    }
    
    if (!process_file(argv[1])) {
        fprintf(stderr, "Ошибка обработки файла\n");
        return EXIT_FAILURE;
    }
    
    printf("Успешно завершено\n");
    return EXIT_SUCCESS;  // Возврат 0
}
```

## Сравнение механизмов завершения

| Механизм | `atexit` | Потоки | Буферы | Сигнал |
|----------|----------|--------|--------|--------|
| `return` из `main` | ✅ Вызывает | ✅ Закрывает | ✅ Сбрасывает | Нет |
| `exit()` | ✅ Вызывает | ✅ Закрывает | ✅ Сбрасывает | Нет |
| `_Exit()` | ❌ Не вызывает | ⚠️ Зависит | ⚠️ Зависит | Нет |
| `quick_exit()` | ⚠️ `at_quick_exit` | ❌ Не закрывает | ❌ Не сбрасывает | Нет |
| `abort()` | ❌ Не вызывает | ⚠️ Зависит | ⚠️ Зависит | SIGABRT |

## Пример: полное завершение

```c
#include <stdio.h>
#include <stdlib.h>

void cleanup1(void) {
    printf("Очистка 1\n");
}

void cleanup2(void) {
    printf("Очистка 2\n");
}

int main(void) {
    // Регистрация обработчиков
    atexit(cleanup1);
    atexit(cleanup2);
    
    printf("Основная программа\n");
    
    // Возврат из main запускает:
    // 1. cleanup2 (последний зарегистрирован → первый вызван)
    // 2. cleanup1
    // 3. Закрытие потоков
    // 4. Возврат в ОС
    return 0;
}
```

**Вывод:**
```
Основная программа
Очистка 2
Очистка 1
```
