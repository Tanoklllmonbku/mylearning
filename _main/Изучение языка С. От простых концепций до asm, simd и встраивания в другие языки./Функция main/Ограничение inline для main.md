# Ограничение `inline` для функции `main`

## Запрет на использование `inline`

Функция `main` **не может** быть объявлена с ключевым словом `inline`:

```c
// ❌ ЗАПРЕЩЕНО: main не может быть inline
inline int main(int argc, char *argv[]) {
    return 0;
}
```

## Причины ограничения

### 1. Точка входа программы

`main` — это **точка входа**, куда среда выполнения передаёт управление при запуске программы. Среда выполнения ожидает:
- Существование адреса функции `main` для передачи управления
- Возможность вызова `main` как обычной функции

```c
// Среда выполнения делает что-то вроде:
extern int main(int, char**);
int result = main(argc, argv);  // Требуется адрес функции
```

### 2. Семантика `inline`

Ключевое слово `inline` в C означает:
- **Подсказка** компилятору для встраивания функции в место вызова
- Возможность **множественных определений** в разных трансляционных единицах
- Функция может не иметь **единого адреса** в памяти

```c
// inline-функция может быть встроена в код вызывающего:
inline int add(int a, int b) { return a + b; }

// Вызов может быть заменён на код функции:
int x = add(2, 3);  // → int x = 2 + 3;
```

### 3. Конфликт семантик

| Требование | `main` | `inline` |
|------------|--------|----------|
| Единый адрес | ✅ Обязательно | ⚠️ Не гарантирован |
| Вызов средой выполнения | ✅ Требуется | ❌ Не предполагается |
| Встраивание в код | ❌ Невозможно | ✅ Предполагается |
| Множественные определения | ❌ Запрещено | ✅ Разрешено |

## Технические последствия

### Отсутствие адреса функции

Для `main` **запрещено получение адреса**:

```c
// ❌ ЗАПРЕЩЕНО для main:
int (*fp)(int, char**) = &main;  // Ошибка компиляции
```

Если бы `main` была `inline`, компилятор мог бы:
- Не выделять отдельный блок кода для функции
- Встраивать код непосредственно в точки вызова
- Сделать невозможным получение корректного адреса

### Среда выполнения (runtime)

Среда выполнения C (CRT — C Runtime) полагается на:
1. Существование **адреса** `main` в таблице символов
2. Возможность **передать управление** по этому адресу
3. **Единое определение** `main` в программе

```asm
; Псевдокод среды выполнения:
call _main    ; Требуется адрес main для вызова
```

## Стандарт C

Стандарт C явно определяет `main` как функцию с **определением**, а не как кандидата на встраивание:

> «A program in a hosted environment shall contain a **definition** of a function named `main`»

Слово **definition** подразумевает:
- Выделение памяти для кода функции
- Существование адреса для вызова
- Невозможность встраивания как замены определения

## Сравнение с обычными функциями

| Функция | `inline` разрешён | Причина |
|---------|-------------------|---------|
| `main` | ❌ Нет | Точка входа, требуется адрес |
| Пользовательские функции | ✅ Да | Обычный вызов из кода |
| Функции в заголовках | ✅ Да | Множественные включения |

## Пример ошибки компиляции

```c
// main.c
inline int main(int argc, char *argv[]) {
    return 0;
}
```

**GCC/Clang:**
```
error: cannot inline main function
```

**MSVC:**
```
error C2231: cannot take address of 'main'
```

## Вывод

Ограничение на `inline` для `main` является фундаментальным требованием стандарта C, обусловленным:
1. Ролью `main` как точки входа программы
2. Необходимостью существования адреса функции для среды выполнения
3. Конфликтом между семантикой `inline` и требованиями к точке входа
